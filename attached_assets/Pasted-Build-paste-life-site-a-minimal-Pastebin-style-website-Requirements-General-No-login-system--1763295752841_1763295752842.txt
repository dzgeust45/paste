General

Features

Create paste

Fields: title (optional), content (required), language (optional), expiration (1h / 1d / 1w / never), privacy (public, unlisted, private).

Return a short slug and a secret_token for edit/delete.

Show copyable link + raw link.

View paste

Syntax highlighting.

/raw/<slug> (text/plain).

Show created_at, expires_at, views.

Expired pastes return a friendly “expired” message.

Edit/Delete paste

No login — must require secret_token.

Must be handled server-side using secure Edge Function or backend logic with service role key.

Privacy

public: listable/searchable.

unlisted: only via URL, not searchable.

private: only accessible by secret token (Edge Function check).

Extra

Paste statistics (views).

Raw + embed snippet (<iframe>).

Optional CAPTCHA support.

Per-IP rate limiting to prevent abuse.

Security

Prevent XSS (sanitize output or render inside code block).

Use Supabase Row Level Security:

anon users can INSERT

anon users can SELECT only public/unlisted

private pastes only selectable via server/Edge Function

Never expose service role key in frontend.

Database schema

Use this exact table schema:

create table public.pastes (
  id uuid default gen_random_uuid() primary key,
  slug text not null unique,
  title text,
  content text not null,
  language text,
  privacy text not null default 'unlisted',
  secret_token text not null,
  created_at timestamptz default now(),
  expires_at timestamptz,
  views int default 0
);

create index on public.pastes (created_at desc);
create index on public.pastes using gin ((to_tsvector('english', coalesce(title,'') || ' ' || content)));
